组件系统

组件系统是Extag的核心，其他模块和类的设计几乎都是在为组件的实现做铺垫。组件主要是对一般标签元素的扩展和加强，支持模板、数据绑定、属性验证等高级功能。

组件，元素，片段，文本

组件Component、元素Element、片段Fragment和文本Text，共同构成了Shell层。文本是最简单的，元素则可以设置样式、插入或移除子节点等，片段则只负责管理一组子节点。一般来说，使用组件和模板后，就不需要手动创建文本、元素、片段或子组件。下面的代码展示了相关的API，移步这里可以看到结果。

// 定义组件
var Panel = Extag.defineClass({
  // constuctor: function() { Extag.Component.apply(this, arguments) },
  extends: Extag.Component,
  statics: {
    template:'<div class="panel" x:style="border: solid 2px @{color};">' +
                '<h3 class="head" x:style="background-color: @{color};">' + 
                    '@{title}' +
                '</h3>' +
                '<div class="body">' + 
                    '<x:slot></x:slot>' + 
                '</div>' +
            '</div>',
    attributes: ['title', 'color']
  }
});

// 创建组件
var panel = Extag.Component.create(Panel);

var rect = Extag.Element.create('svg:rect');

var text = Extag.Text.create('');

// appendChild, insertChild, removeChild, replaceChild
svg.appendChild(rect);

Text、Element和Fragment作为final class都是不能被继承的，而Component则具有很好的可扩展性。 这里使用了Extag的工具方法defineClass，extends表示继承，statics表示一些静态属性，constrcutor表示自定义构造函数。你依然可以用你喜欢的方式来定义一个子组件类，例如使用ES6的class语法糖。重要的是，在子类构造函数中要调用父类Component的构造函数。

props, attrs, style, classes

无论是组件、元素还是文本，都可使用set方法更新它的DOM属性值，或通过assign方法多次调用set。某个DOM属性的值发生更改时，这个DOM属性就会被标记为脏，并触发失效机制，等待渲染。异步渲染过后，脏标记被自动清除。

对于组件和元素，它们的成员对象style、attrs和classes，都有脏标记的功能，并与失效机制挂钩。因为我们可以在模板使用数据绑定，所以一般不需要手动更新style、attrs和classes的属性值。

props v.s. attrs

对于DOM，常用的property一般都是基本类型的，且都有对应的attribute， 例如checked、title、src等。但也有一些常用的property是没有对应的attribute的，例如innerHTML、textContent等。还有一些property，虽有对应的attribute，但它们本身是object，而非字符串，例如SVG元素circle的属性cx、cy等。

为了应对这种不一致的情况，在渲染props时，Extag采取的策略是，对于那些重要的property，如checked、innerHTML，我们直接对其赋值，否则使用setAttribute方法进行赋值。

对于data-*之类的自定义attribute，建议使用组件、元素的成员attrs来管理。

Shell层通过ExtagDom与真实DOM层结合后，文本、元素和以元素为根的组件的属性$skin都指向实际的DOM节点。而片段和以片段为根的组件的$skin则为null，因而设置它们的DOM属性等，并没有什么实质的作用。

组件自定义特性

对于组件，除了设置DOM属性外，还可以有自定义属性，可自定义属性的默认值，还可为自定义属性添加一些描述。我们称这些自定义的属性为组件特性，它们默认是bindable的。在创建这一组件的首个实例时，Extag就会在它的prototype上为每个特性创建一对getter和setter。 对于这样自定义的属性，当它的值发生改变时，会派发属性更改通知事件，并触发失效机制。

var Demo = Extag.defineClass({
  extends: Extag.Component,
  statics: {
    template: '<div></div>',
    attributes: {
      label: '',
      amount: 0,
      active: true,
      date1: Extag.anew(Date, '2020/01/01'), // 用一个构造函数和参数列表来生成默认值
      date2: Extag.inst(function() { return new Date(); }) // 用一个函数来生成默认值
    } 
    // 或者attributes: ['label', 'active', 'amount', 'date1', 'date2']，默认值都为undefined
  }
});
​
var demo = Extag.Component.create(Demo);
demo.label = 'abcd';    // 等同于demo.set('label', 'abcd')，派发事件`changed.label`;
demo.active = false;    // 等同于demo.set('active', false)，派发事件`changed.active`;
demo.amount = 20000;    // 等同于demo.set('amount', 20000)，派发事件`changed.amount`;
demo.undeclared = 1;    // 不同于demo.set('undeclared', 1)，因为未声明为特性;

如果所有的特性都不需要默认值，attributes可定义为一个特性名数组。如果某个特性的默认值不是基础类型的，则可用Extag.anew或Extag.inst为其生成默认值。

如果不希望自定义属性是bindable的，但又希望它在组件模板中可用，则可以这样定义：

attributes: {
  filter: {
    type: Function,
    bindable: false
  }
}

声明为不可绑定的属性，Extag不会为之创建getter/setter，自然也不会派发属性更改通知事件。

注意，不同于DOM特性，组件attributes中自定义的属性，是没有脏标记功能的，故而不会被渲染到真实DOM层。如果你在attributes中定义了某个已有的DOM属性，例如title，它是不会被渲染到组件根元素上的。所以，最好避免在组件attributes中定义已有的DOM属性。

另外，组件特性的点语法赋值和set方法设置属性值的关系，可类比DOM属性赋值和setAttribute方法之间的关系。

组件特性验证

在开发环境版本的Extag中，组件特性是支持验证的。你可以指明特性的类型，还可以提供一个验证函数，以实现自定义验证的需要。为此，我们可以在子组件类的静态属性attributes中这样进行定义：

attributes: {
  // 对于boolean、number、string类型
  label: {
    type: 'string'
  },
  amount: {
    type: 'number',
    value: 0 // 提供默认值
  },
  active: {
    type: 'boolean'
  },
​
  // 对于Object类型，使用它们的构造函数
  list: {
    type: Array
  }, 
  tool: {
    type: Knife
  },
​
  // 以上是简单的类型验证，亦可允许多类型，模式匹配或自定义一个验证函数
  date: {
    type: ['number', 'string', Date]
  },
  name: {
    validator: /^[a-zA-Z0-9]{1,32}$/,
  },
  score: {
    type: 'number',
    validator: function(value) {
      if (value < 0 || value > 100) {
        return 'out of the range 0~100'; // 验证不通过，则返回一段错误描述
      }
    }
  }
}

验证失败后，开发环境版本的Extag会在控制台中输出一条警告。生产环境版本的Extag，默认不执行验证过程，但你仍然可以访问Extag.Validator，手动调用它的validate方法或troubleShoot方法，并且不会在控制台输出警告信息。validate方法只返回true（验证成功）或false（验证失败），而troubleShoot方法验证失败的时候返回一段错误描述。

var Demo = Extag.defineClass({
    extends: Extag.Component,
    statics: {
      template: '<div></div>',
      attributes: {
        score: {
          validator: function(value) {
            if (typeof value != 'number') {
              return 'not a number';
            }
            if (value < 0 || value > 100) {
              return 'out of the range 0~100';
            }
          }
        }
      }
    },
    setup: function() {
      this.on('changed.score', this.onChangedScore.bind(this));
    },
    onChangedScore: function() {
      var error = Extag.Validator.troubleShoot(this, 'score', this.score); 
      if (error) {
        // this.emit('invalid.score', error);
        console.log('invalid score, ' + error);
      }
    }
})

组件特性拦截

组件特性默认是bindable的，本身就有一对getter/setter，而你可以进一步定义自己的get和set方法，来拦截原有默认的取值、赋值的操作。你可以在自定义的get方法中覆盖默认的返回值，可以在自定义的set方法中追加一些操作，等等。例如：

var Demo = Extag.defineClass({
  extends: Extag.Component,
  statics: {
    template: '<div></div>',
    attributes: {
      score: {
        get: function(props) {
          return props.score; // 默认返回的就是props.score
        },
        set: function(value, props) {
          if (typeof value === 'number' && value >= 0 && value <= 100) {
            props.score = value; // props是所有特性值真正存放的地方
          }
        }
      }
    }
  }
});
​
var demo = Extag.Component.create(Demo);
demo.on('changed.score', function() {
  console.log('score', demo.score);
});
demo.score = 50;        // 输出：score 50
demo.score = '0';       // 没有输出
demo.score = -50;       // 没有输出
console.log(demo.score) // 输出：50

组件特性依赖

你可以定义一个依赖于其他特性的特性，它的值是由其他特性值计算得到的。对于模板中复杂而冗长的绑定表达式，你就可以使用一个依赖特性来代替。

var Demo = Extag.defineClass({
  extends: Extag.Component,
  statics: {
    template: '<h2>@{fullName}</h2>',
    attributes: {
      firstName: '',
      lastName: '',
      fullName: {
        get: function() {
          return this.firstName + ' ' + this.lastName;
        }
      }
    }
  }
});

就比如上面代码展示的，`fullName`依赖于`fistName`和`lastName`。只要`firstName`或`lastName`的值发生改变，就会重新将`fullName`渲染到页面。

组件内部状态

组件可能含有自己的内部状态，是外界不可修改的。组件默认不含有成员对象state，你可以根据组件的特点，以不同的方式，在组件prototype或attributes中声明state（或者其他名字），在模板绑定表达式中使用state，在setup时设置初始state。

你可以使用Extag提供的Store类型对象作为state，调用它的set或assign方法，在它的属性值发生更改时可派发changed类型事件。

var Demo = Extag.defineClass({
  extends: Extag.Component,
  statics: {
    template: '<div>@{state.seconds}</div>'
  },
  setup: function() {
    // setup发生在模板引擎工作之前。在这里定义this.state，以便模板引擎能找到state。
    var state = this.state = new Extag.Store({seconds: 0});
    setInterval(function() {
      state.set('seconds', state.seconds + 1);
    }, 1000);
  }
});

如果state整体是易变的，你也可以使用普通的Object，更改状态时调用一下invalidate方法。同时，在数据绑定表示式末尾追加符号^，则在组件进入update阶段的时候，派发update事件就会执行state相关的绑定表达式。

var Demo = Extag.defineClass({
  extends: Extag.Component,
  statics: {
    template: '<div>' +
                // 组件update阶段，会派发update事件，触发以下绑定。
                '<span>x: @{state.x ^}</span>, ' +
                '<span>y: @{state.y ^}</span>, ' +
                '<span>z: @{state.z ^}</span>, ' +
              '</div>'
  },
  setup: function() {
    // setup发生在模板引擎工作之前。在这里定义this.state，以便模板引擎能找到state。
    this.state = {x: 0, y: 0, z: 0};
  },
  setState: function(someState) {
    Object.assign(this.state, someState); // 新状态合并入当前状态
    this.invalidate(); // 触发组件的失效机制，而后可进入异步的update阶段，派发update事件
  }
});

组件contents

组件在内部管理自己的children，在外部接受contents。在模板中，你可以使用x:slot标签来自动收集匹配的contents。本章第一节的代码中，Panel模板就使用了一个x:slot，具体用法参见模板语法的组件slots。或者不使用x:slot，你也可以选择在何处手动插入来自外部的contents。

// ...
var Panel = Extag.defineClass({
  extends: Extag.Component,
  statics: {
    attributes: ['title', 'color'],
    template:'<div class="panel" x:style="border#: solid 2px @{color};">' +
                '<h3 class="head" x:style="background-color@: color;">@{ title }</h3>' +
                '<div class="body" x:name="slot"></div>' +
             '</div>'
  },
  onInited: function() {
    this.on('changed.contents', (function() {
      this.slot.setChildren(this.contents);
    }).bind(this));
  }
});
​

panel.set('contents', [svg, span]);

组件边界处理

在将一个组件用作其他组件的子组件时，这个子组件的根元素就是内外边界结合的地方。为了避免边界冲突，组件内部只应在模板根元素上声明class和一些DOM事件，style等其他DOM属性应一律留在外部操作（用作子组件时）。否则一不注意，就可能引发冲突，造成混乱。

如果不想接受外部的DOM属性、样式的直接修改，那么你可以在子组件模板中使用<x:frag>作为根元素。即组件以Fragment的形式出现，把真正的根元素作为<x:frag>的子节点。这样在外部，则必须通过组件声明的attributes，才能够间接地修改到实际根元素的样式等DOM属性。

<x:frag>
  <div x:style="color@: color;">...</div>
</x:frag>

有了<x:frag>，你可以向其中添加更多的子节点。渲染时，<x:frag>是不存在的，它所在的位置将被所有子节点所占据。

组件生命周期

create->(steup)->initialize->(onInited)->attach->(onAttached)->invalidate->update->(onUpdating)->render->(onRendered)->detach->(onDetached)->destroy->(onDestroyed)

create: 创建某种类型组件的首个实例时，首先要在prototype上定义声明的attributes，编译声明的模板。同时，对于该类型组件的所有实例，还需要设置attributes的默认值，定义好props, attrs, style 和 classes等成员。

initialize: 这一阶段，有了编译好的模板，就会启动模板引擎，为组件添加子内容、事件处理器和数据绑定等等。在此之前，你可以在setup方法中预先完成些事，比如初始化state，将事件处理函数绑定到this。之后则会触发onInited钩子函数，你可以补充一些事情，比如监听window的事件。

attach: 对于以元素为根的组件，你可以调用attach方法，将之附着到页面上已有的DOM元素上（要求该DOM元素与组件根元素的tagName和namespace一致）。进入渲染阶段，根组件的后代元素会由上而下，跟着调用各自的attach方法，附着到通过ExtagDom创建的DOM元素上。以元素为根的组件attach之后就会触发onAttached钩子函数，其接受的第一个参数就是实际附着的DOM元素（注意，这时候的DOM元素还没有children）。以片段为根的组件调用attach方法无效，不会触发onAttached钩子函数。

invalidate: 当组件、元素的属性、样式发生变化后，组件、元素就会触发失效机制，进入失效状态，需要等待下一刻把变动的部分渲染到DOM层，才会恢复到正常状态。

update: 这时候将派发update事件，导致一系列数据绑定执行生效，更多的组件和元素触发失效机制。在此期间，将触发onUpdating钩子函数，这是你在render到DOM层之前，修改数据的最后一次机会。

render: 这一阶段，就是通过ExtagDom把Shell层变动的部分，自上而下依次渲染到DOM层。之后，对于以元素为根的组件，则自下而上触发onRendered钩子函数，其接受的第一个参数就是实际附着的DOM元素，且这时候的DOM元素已包含children。

detach: DOM元素从文档中移除后，Shell层的元素、组件的detach方法就会被调用，从而脱离已附着的DOM元素。后代元素、组件也会跟着调用各自的detach方法，脱离已附着的DOM元素。之后，对于以元素为根的组件，就会触发onDetached钩子函数，其接受的第一个参数就是之前附着过的DOM元素。

destroy: detach之后, destroy就被自动调用，将要清理children，移除所有的数据绑定和事件监听。之后触发onDestroyed钩子函数，你可以在这里清理遗留的东西，如对window事件的监听。

其中，attach, detach, update, render, invalidate都是Component对象的实例方法，大部分情况下由Extag内部自主调用，应避免在继承时重写这些方法。
